# Separate from venv_ltx (LTX-0.9.7-distilled): LTX-2.3's pipeline
# unconditionally imports Gemma3ForConditionalGeneration (its default text
# encoder) at module load time. That needs transformers>=~4.50, but
# venv_ltx pins transformers<4.50 specifically to dodge a DIFFERENT
# tokenizer regression that breaks LTX-0.9.7-distilled (see
# requirements/ltx.txt). Confirmed via a real ImportError in Colab -- not
# assumed. Install transformers from git here too, same reasoning as
# requirements/qwen.txt: guarantees current model support rather than
# guessing a version boundary.
